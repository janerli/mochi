# mochi 🍡

Личный таск-трекер с отдельным разделом заметок. Desktop (Electron) и web
делят один и тот же React-код — Electron открывает тот же UI, что и браузер,
указывая на один и тот же захостенный сервер. Данные живут в workspace — у
каждого аккаунта есть личный (создаётся автоматически) и можно создавать/
присоединяться к общим — задачи и заметки внутри одного workspace видны и
редактируются всеми его участниками, живой синк — через Socket.io.

## Стек

- `server/` — Fastify + Prisma + **PostgreSQL (Supabase)** + Socket.io +
  JWT-авторизация (httpOnly-кука). Файловые вложения — в **Supabase Storage**
  (не на локальном диске — на бесплатном тарифе Render нет постоянного диска)
- `apps/web/` — Vite + React + TypeScript, обычный CSS (без Tailwind), TanStack Query
- `apps/desktop/` — тонкая Electron-обёртка: просто открывает URL захостенного
  сервера в окне, никакого встроенного бэкенда

Сервер разворачивается один раз на **Render** и обслуживает и web (раздаёт
собранный `apps/web/dist` с того же origin — см. `STATIC_DIR`), и desktop —
оба клиента ходят в один и тот же бэкенд, поэтому синк работает по-настоящему,
между любыми устройствами, а не только на одной машине.

## Функционал

- Доска задач: три колонки, drag-and-drop карточек между ними (плюс кнопки
  "переместить" для доступности), приоритеты, фильтр по тегам, редактирование,
  файловые вложения (до 20 МБ на файл, Supabase Storage), управление тегами
  (✎ рядом с фильтром — переименовать или очистить тег сразу у всех задач в
  workspace, без отдельной таблицы тегов)
- Время задач: дедлайн с точным временем (не только дата), оценка времени на
  выполнение, повторяющиеся задачи (ежедневно/еженедельно/ежемесячно —
  при отметке "готово" следующая копия создаётся автоматически), напоминания
  (в момент дедлайна / за 15 мин / 30 мин / час / день) — нативное уведомление
  ОС через стандартный Notification API, чисто на клиенте, без пуш-сервера:
  работает, пока открыто приложение (вкладка браузера или desktop-окно)
- Заметки — два типа: быстрые (простой текст, карточка-стикер) и большие
  (модалка на весь экран, markdown-тулбар, чек-листы). Обе: цвет (у быстрых),
  группы/категории с фильтром, закрепление, привязка к задаче
- Поиск по задачам и заметкам в шапке
- Календарь: месячная сетка с задачами по дедлайнам (с точным временем)
- Фокус-сессия: помодоро-таймер (25/5 минут), завершённые сессии сохраняются
  на сервере — счётчик переживает перезагрузку и синхронизируется между
  устройствами одного аккаунта
- Аккаунт: регистрация/вход по почте и паролю. Сброс пароля без почты — код
  восстановления (4×5 символов) показывается один раз при регистрации, им же
  можно сбросить пароль, если забыл(а) — код при этом ротируется. Смена пароля
  и перевыпуск кода — в настройках (⚙️ в шапке)
- Workspace: переключатель в сайдбаре, у каждого аккаунта — личный (нельзя
  удалить) + сколько угодно общих. Приглашение — по короткому коду (владелец
  генерирует в настройках workspace, делится любым способом — код не
  привязан к почте). Владелец может переименовать, перевыпустить код,
  удалить workspace целиком; участник может выйти. Список участников виден всем

## Настройка с нуля

### 1. Supabase (база данных + хранилище файлов)

1. Зарегистрируйся на [supabase.com](https://supabase.com), создай новый проект (Free tier).
2. **Database → Connection string** — скопируй два варианта: *Transaction* (порт 6543,
   с `?pgbouncer=true`) и *Session*/*Direct* (порт 5432). Впиши их как
   `DATABASE_URL` и `DIRECT_URL` в `server/.env` (см. `server/.env.example`).
3. **Storage** → создай новый **приватный** бакет с именем `attachments`
   (без публичных policy — доступ проверяется на уровне сервера по workspace,
   не через Supabase RLS).
4. **Settings → API** — скопируй `Project URL` → `SUPABASE_URL` и ключ
   `service_role` (не `anon`!) → `SUPABASE_SERVICE_KEY` в `server/.env`.
5. Накати схему на свежую базу:
   ```bash
   cd server && npx prisma migrate deploy
   ```

### 2. Локальная разработка

```bash
npm install
npm run dev           # сервер (порт 4000) + web (порт 5173), общая Supabase-база
npm run dev:desktop   # то же самое + Electron-окно (тоже смотрит на localhost:5173)
```

Дев-окружение и продакшен на Render используют **одну и ту же** Supabase-базу
(это личный проект на одного человека — заводить вторую базу под dev пока
избыточно). Просто аккуратнее с миграциями на общей базе.

### 3. Деплой сервера на Render

1. Запушь репозиторий на GitHub (`git remote add origin ...`, `git push -u origin master`).
2. На [render.com](https://render.com) → **New → Blueprint**, укажи репозиторий —
   подхватит `render.yaml` из корня и создаст веб-сервис `mochi-server`
   автоматически (build/start команды уже прописаны).
   Либо вручную: **New → Web Service**, Node runtime, Free plan,
   build command `npm install && npm run build -w apps/web && npm run build -w server`,
   start command `npm run db:deploy -w server && npm run start -w server`.
3. В **Environment** пропиши те же переменные, что в `server/.env`
   (`DATABASE_URL`, `DIRECT_URL`, `SUPABASE_URL`, `SUPABASE_SERVICE_KEY`,
   `SUPABASE_BUCKET=attachments`) + `NODE_ENV=production`. `JWT_SECRET` и
   `STATIC_DIR` уже заданы в `render.yaml`.
4. После первого деплоя скопируй выданный URL (`https://mochi-server.onrender.com`
   или похожий) — открой его в браузере, там должен быть весь mochi.

Бесплатный тариф Render засыпает после 15 минут простоя — первое открытие
после паузы будет с задержкой (~30–50 сек), это нормально для личного проекта.

### 4. Сборка десктоп-инсталлятора

```bash
npm run installer -w apps/desktop
```

`apps/desktop/electron/main.js` открывает `MOCHI_SERVER_URL` (по умолчанию —
константа `PRODUCTION_SERVER_URL` в файле; обнови её на реальный URL из шага 3
перед сборкой). Инсталлятор — `apps/desktop/release/Mochi Setup <версия>.exe`.
Для быстрой проверки без NSIS: `npm run dist -w apps/desktop`.

Иконка (`apps/desktop/build/icon.ico` + `icon.png`) — кавайное личико-моти,
вшита и в `Mochi.exe`, и в инсталлятор, и в окно приложения.
