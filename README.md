# mochi 🍡

Личный таск-трекер с отдельным разделом заметок. Desktop (Electron) и web
делят один и тот же React-код — Electron просто открывает тот же UI, что и
браузер. Данные живут в workspace — у каждого аккаунта есть личный (создаётся
автоматически) и можно создавать/присоединяться к общим — задачи и заметки
внутри одного workspace видны и редактируются всеми его участниками, живой
синк — через Socket.io.

**Важно про синк веб↔desktop**: он работает, только пока оба клиента стучатся
в один и тот же запущенный сервер. В деве (`npm run dev` + `npm run dev:desktop`)
это так по умолчанию — один процесс на порту 4000. У **собранного инсталлятора**
всё иначе: чтобы приложение запускалось без терминала, Electron поднимает
сервер прямо у себя внутри (порт 4127, только `127.0.0.1`) со своей локальной
базой (`%APPDATA%/Mochi/mochi.db`) — это отдельный остров, изолированный от
`server/prisma/dev.db`. Если одновременно открыт `localhost:5173` (dev) и
установленный `Mochi.exe` — это два разных приложения с разными данными,
между ними ничего не синкается. Синк между несколькими устройствами (не одна
машина) вообще не реализован — для этого нужен отдельный всегда работающий
сервер (свой хостинг), которого пока нет.

## Стек

- `server/` — Fastify + Prisma + SQLite + Socket.io + JWT-авторизация (httpOnly-кука)
- `apps/web/` — Vite + React + TypeScript, обычный CSS (без Tailwind), TanStack Query
- `apps/desktop/` — Electron-обёртка над `apps/web`; в собранном инсталляторе сам
  бэкенд запускается внутри Electron (см. «Сборка десктоп-инсталлятора» ниже) —
  двойной клик и никакого терминала

## Функционал

- Доска задач: три колонки, drag-and-drop карточек между ними (плюс кнопки
  "переместить" для доступности), приоритеты, фильтр по тегам, редактирование,
  файловые вложения (до 20 МБ на файл, хранятся на диске — `server/uploads/`
  в деве), управление тегами (✎ рядом с фильтром — переименовать или очистить
  тег сразу у всех задач в workspace, без отдельной таблицы тегов)
- Время задач: дедлайн с точным временем (не только дата), оценка времени на
  выполнение, повторяющиеся задачи (ежедневно/еженедельно/ежемесячно —
  при отметке "готово" следующая копия создаётся автоматически), напоминания
  (в момент дедлайна / за 15 мин / 30 мин / час / день) — нативное уведомление
  ОС через стандартный Notification API, чисто на клиенте, без пуш-сервера:
  работает, пока открыто приложение (вкладка браузера или desktop-окно)
- Заметки — два типа: быстрые (простой текст, карточка-стикер) и большие
  (модалка на весь экран, markdown-тулбар, чек-листы). Обе: цвет (у быстрых),
  группы/категории с фильтром, закрепление, привязка к задаче
- Поиск по задачам и заметкам в шапке
- Календарь: месячная сетка с задачами по дедлайнам (с точным временем)
- Фокус-сессия: помодоро-таймер (25/5 минут), завершённые сессии сохраняются
  на сервере — счётчик переживает перезагрузку и синхронизируется между
  устройствами одного аккаунта
- Аккаунт: регистрация/вход по почте и паролю. Сброс пароля без почты — код
  восстановления (4×5 символов) показывается один раз при регистрации, им же
  можно сбросить пароль, если забыл(а) — код при этом ротируется. Смена пароля
  и перевыпуск кода — в настройках (⚙️ в шапке)
- Workspace: переключатель в сайдбаре, у каждого аккаунта — личный (нельзя
  удалить) + сколько угодно общих. Приглашение — по короткому коду (владелец
  генерирует в настройках workspace, делится любым способом — код не
  привязан к почте). Владелец может переименовать, перевыпустить код,
  удалить workspace целиком; участник может выйти. Список участников виден всем

## Первый запуск (разработка)

```bash
npm install
npm run db:migrate   # создаёт server/prisma/dev.db и накатывает схему
```

```bash
npm run dev           # сервер (порт 4000) + web (порт 5173)
npm run dev:desktop   # то же самое + Electron-окно
```

Открой http://localhost:5173, зарегистрируйся — задачи и заметки появятся
и в Electron-окне под тем же аккаунтом без перезагрузки (и наоборот).

## Сборка десктоп-инсталлятора

```bash
npm run installer -w apps/desktop
```

Соберёт `server` и `apps/web`, сгенерирует пустую примигрированную SQLite-базу
(`template.db`) и упакует всё в `apps/desktop/release/Mochi Setup <версия>.exe`.

Как это устроено: в собранном приложении Electron сам поднимает встроенный
Fastify-сервер (порт `4127`, только на `127.0.0.1`) внутри своего основного
процесса — никакого отдельного `node`/терминала не нужно. При первом запуске
`template.db` копируется в `%APPDATA%/Mochi/mochi.db`, там же генерируется и
сохраняется случайный JWT-секрет (`secret.txt`), поэтому сессия переживает
перезапуск приложения. Библиотека `@fastify/static` отдаёт собранный
`apps/web/dist` с того же порта — API и фронтенд на одном origin, без прокси.

Для быстрой проверки без полной сборки инсталлятора:
`npm run dist -w apps/desktop` — то же самое, но без шага NSIS
(результат в `apps/desktop/release/win-unpacked/Mochi.exe`).

Иконка (`apps/desktop/build/icon.ico` + `icon.png`) — кавайное личико-моти,
сгенерирована из `.logo-mark`-дизайна и вшита и в `Mochi.exe`, и в сам
инсталлятор, и в окно приложения.

Обновления схемы БД: у существующего пользователя `template.db` не
перезаписывает уже созданный `mochi.db` — вместо этого при каждом старте
`server/src/migrate.ts` доигрывает недостающие миграции (используя те же
файлы, что и `prisma migrate`, без переноса всего Prisma CLI в инсталлятор).
Так апдейт с новой колонкой в схеме не ломает данные тех, кто уже пользуется
приложением.

## Дальше по плану

- Обновление Prisma 5 → 7 (мажорный апгрейд, пока не критично)
- Собранный инсталлятор снова нужно пересобрать (`npm run installer -w apps/desktop`)
  — добавилась таблица вложений, схема БД поменялась (аддитивно, `migrate.ts`
  доиграет без проблем)
- Настоящий синк между несколькими устройствами (см. предупреждение в начале
  файла) — нужен отдельный всегда работающий сервер, а не только встроенный
  в desktop-приложение
- `server/src/migrate.ts` пока не умеет доигрывать структурные скачки схемы
  (переход на workspace добавил новые обязательные поля/таблицы, а не только
  колонки) — на реальных пользователях это не встречалось (никто ещё не
  устанавливал версию до workspace), но если когда-нибудь понадобится апгрейд
  через настолько большой скачок схемы, миграцию придётся дописывать руками
